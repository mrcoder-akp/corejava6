What Is Compilation?

Compilation is the process in which source code is converted to executable code by a compiler. This code is stored in a separate file. In Java, this file is called the class file. The execution name of the file will be class.
:Genrating a executable code from the source code is called compilation.

1.Parsing

Checking error inside source code is called parsing.

2.Translation

If source cdode has no error compiler convert source code into machine understadable code this is called translation.

What Is a Compiler?

The Java Compiler takes source code, written in a text file, and converts it into bytecode. Examples of compilers are Eclipse, IntelliJ IDEA, JDeveloper, BlueJ, etc.

Tools

Notepad, Net Bean, Eclipse, etc.

Runtime

Class-File → ClassLoader → Bytecode → Interpreter → Runtime → Hardware

Compile and Execute Class

If you are using Notepad to save the class:

  • Step 1 → javac Filename.java (Compilation done).
  • Step 2 → java Filename (File Execute).

To see all methods:

  • Step 1 → javap Filename

Is it mandatory to save the source file with the class name?

If the class is public, the source file must be saved with the same name as the class. If the class is not public, it is not necessary to save the file with the class name. However, if you are saving files with different names, you will need to check the class file's name to find the corresponding source file.

When a public class is created, it must be saved with the class name. If the class name and file name are the same, background compilation is easier and works smoothly.

For example, if two classes, AA and BB, are created and inherited into a third class, the background compilation will work seamlessly because the class and file names for AA and BB match. This helps in easier background compilation.
