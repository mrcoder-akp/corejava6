Without keywords, you cannot write the source code of your program. Keywords are the vocabulary of a programming language. Java provides up to 60 keywords. Some of them are:

  • class
  • interface
  • public
  • protected
  • private
  • static
  • int
  • boolean
  • return
  • void
  • if
  • abstract, assert, boolean, break, byte, case, catch, char, class, const
  • continue, default, do, double, else, enum, extends, final, finally, float
  • for, goto, if, implements, import, instanceof, int, interface, long, native
  • new, package, private, protected, public, return, short, static, strictfp, switch
  • synchronized, this, throw, throws, transient, try, void, volatile

    • Explanation...

  • abstract: Used to declare a class or method that must be implemented by subclasses.
  • assert: Used to test assumptions in the code during runtime.
  • boolean: A data type that holds either true or false.
  • break: Exits a loop or switch statement.
  • byte: A data type that holds an 8-bit integer.
  • case: Defines a branch in a switch statement.
  • catch: Handles exceptions thrown in a try block.
  • char: A data type for a single character.
  • class: Declares a class.
  • const: Reserved but not used in Java (use final instead).
  • continue: Skips the current iteration of a loop and goes to the next.
  • default: Defines the default branch in a switch statement.
  • do: Starts a do-while loop that runs at least once.
  • double: A data type for a 64-bit floating-point number.
  • else: Specifies what to do if an if condition is false.
  • enum: Declares a fixed set of constants.
  • extends: Indicates inheritance from a superclass.
  • final: Used to declare constants, prevent inheritance, or prevent method overriding.
  • finally: Runs code after a try-catch block, regardless of the outcome.
  • float: A data type for a 32-bit floating-point number.
  • for: Starts a loop that repeats a block of code.
  • goto: Reserved but not used in Java.
  • if: Tests a condition and executes a block of code if true.
  • implements: Indicates a class implements an interface.
  • import: Includes other Java classes or packages in a program.
  • instanceof: Tests whether an object is an instance of a class or subclass.
  • int: A data type for a 32-bit integer.
  • interface: Declares a blueprint of methods that a class must implement.
  • long: A data type for a 64-bit integer.
  • native: Indicates that a method is implemented in another language like C or C++.
  • new: Creates new objects.
  • package: Groups related classes and interfaces.
  • private: Makes a class member accessible only within its own class.
  • protected: Makes a class member accessible within its package and subclasses.
  • public: Makes a class member accessible from anywhere.
  • return: Exits from a method and optionally returns a value.
  • short: A data type for a 16-bit integer.
  • static: Declares a class-level member shared by all instances.
  • strictfp: Enforces strict floating-point calculations.
  • switch: Starts a multi-branch decision block.
  • synchronized: Prevents multiple threads from accessing a block or method simultaneously.
  • this: Refers to the current object.
  • throw: Used to explicitly throw an exception.
  • throws: Declares exceptions a method might throw.
  • transient: Excludes a field from being serialized.
  • try: Defines a block of code to test for exceptions.
  • void: Specifies that a method does not return a value.
  • volatile: Indicates that a variable may be changed by multiple threads.
