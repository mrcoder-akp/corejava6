Non-Static Class‚ö°

A non-static class in Java is a class defined inside another class and does not have the static keyword. It is also called an Inner Class.
Since a non-static inner class is tied(connected/dependent on) to an instance of the outer class, it cannot exist without an object of the outer class.
‚û°Ô∏è The inner class cannot exist without an object of the outer class.
‚û°Ô∏è To create an object of the inner class, we must first create an object of the outer class.

Example..

class Outer {
    class Inner {
    void display() {
    System.out.println("Inside inner class");
            }
        }
    }
    public class Test {
        public static void main(String[] args) {
            Outer.Inner obj = new Outer().new Inner();
            obj.display();
        }
    }

Why Use a Non-Static Classü§î

To logically group classes together that are only used within a specific class.
To access private members of the outer class.
To make the code more readable and encapsulated.

Types of Non-Static Classesüõ†Ô∏è

There are three types of non-static classes in Java:

1.Member Inner Class
2.Local Inner Class
3.Anonymous Inner Class

Member Inner Class

A member inner class is a normal class defined inside another class without using static. It acts like a normal class but can access all members (even private) of the outer class.

Example..

class Outer {
 private String message = "Hello from Outer class";
    // Non-static Inner class
    class Inner {
        void display() {
        System.out.println(message); // Accessing private member of Outer class
            }
        }
        public static void main(String[] args) {
        Outer outer = new Outer(); // Create Outer class object
        Outer.Inner inner = outer.new Inner(); // Create Inner class object
        inner.display(); // Calling method of Inner class
        }
    }
 output 
 Hello from Outer class

Local Inner Class

A local inner class is defined inside a method of the outer class. It is only accessible within that method.

           }
            }

        inner.show(); // Calling method of Inner class
        }
        public static void main(String[] args) {
            Outer outer = new Outer();
            outer.display();
        }
    }
output 
Inside Local Inner Class

Static Class (Nested Static Class)

In Java, a class itself cannot be static, but we can create a static class inside another class, which is called a static nested class.
A static class (static nested class) is a class inside another class that is declared using the static keyword. Unlike non-static inner classes, a static nested class does not require an object of the outer class to be created.
In Java, a class cannot be static on its own. However, a class inside another class can be made static. This is called a static nested class.
A static nested class is a class inside another class, but it does not require an object of the outer class to be created.
‚úÖ It belongs to the outer class, but it does not depend on it.
‚úÖ It can only access the static members of the outer class.

Example..

class Outer {
  static class Inner {  // Static Nested Class
  void display() {
  System.out.println("Inside Static Nested Class");
}
}
public static void main(String[] args) {
   Outer.Inner obj = new Outer.Inner(); // Creating object of Static Nested Class
    obj.display();
        }
}
output 
Inside Static Nested Class

A static nested class is just like a normal class, but it is defined inside another class and marked as static. Since it is independent of the outer class instance, we do not need to create an object of the outer class. However, if the static nested class contains non-static members (like methods and instance variables), we still need to create an object of the static nested class to access them.

If Everything is Static, No Object is Needed

class Outer {
    static class Inner {  
            static void display() { // Now this method is static
                System.out.println("Inside Static Nested Class");
            }
        }
public static void main(String[] args) {
    Outer.Inner.display(); // No need to create an object
        }
}
 output 
 Inside Static Nested Class

‚úÖStatic nested classes do not require an object of the outer class.
‚úÖIf the static nested class has non-static members, you must create an object of the nested class.
‚úÖIf all members inside the static nested class are static, you can access them directly without an object. 

class Bank {
 private static double interestRate = 5.5; // Static variable (same for all banks)
    // Static Nested Class

     return (amount * interestRate * years) / 100;
            }
    }
public static void main(String[] args) {
    double amount = 10000;
    int years = 3;
    // Calling static method from static nested class
    double interest = Bank.InterestCalculator.calculateInterest(amount, years);
    System.out.println("Interest for " + years + " years on ‚Çπ" + amount + " is: ‚Çπ" + interest);
        }
    }
output

‚úÖ Encapsulation: InterestCalculator is only related to Bank, so we keep it inside Bank.
‚úÖ No Need for Outer Class Object: InterestCalculator does not depend on Bank objects, so it‚Äôs static.
‚úÖ Better Code Organization: Instead of writing InterestCalculator as a separate class, we keep it inside Bank for clarity.
‚úÖ A static nested class is useful when the inner class is logically related to the outer class but does not need access to instance variables of the outer class.
‚úÖIt helps group related functionality inside a class, making the code more organized and readable.

//In a company, employees have ID cards that store their details like name, ID number, and department. The ID Card details are specific to the Employee but do not need access to the Employee's instance variables.
Since an ID Card is logically related to an Employee but does not depend on an Employee object, we can use a static nested class to represent it.
--------------------------------------------------------------------------------------------------------------
class Employee {
        private String name;
        private int empId;
    
        // Constructor for Employee
        Employee(String name, int empId) {
            this.name = name;
            this.empId = empId;
        }
    
        // Static Nested Class for ID Card
        static class IDCard {
            private String department;
            private int idNumber;
    
            // Constructor for IDCard
            IDCard(String department, int idNumber) {
                this.department = department;
                this.idNumber = idNumber;
            }
    
            void displayIDCard() {
                System.out.println("Employee ID Card:");
                System.out.println("Department: " + department);
                System.out.println("ID Number: " + idNumber);
            }
        }
    
        void displayEmployee() {
            System.out.println("Employee Name: " + name);
            System.out.println("Employee ID: " + empId);
        }
    
        public static void main(String[] args) {
            // Creating Employee Object
            Employee emp1 = new Employee("John Doe", 101);
            emp1.displayEmployee();
            
            System.out.println(); // Line Break
            
            // Creating IDCard Object (No need for Employee object)
            Employee.IDCard id1 = new Employee.IDCard("IT", 5001);
            id1.displayIDCard();
        }
    }

Department: IT

Concrete Class

A concrete class is a normal class in Java that contains:
‚úÖ Fields (variables) ‚Äì to store data
‚úÖ Methods (functions) ‚Äì to perform actions
‚úÖ Objects can be created from it ‚Äì unlike abstract or interface classes

Why is it called a "Concrete" Class

The word "concrete" means real or complete.
A concrete class is a fully defined class that provides complete implementation for all its methods.
It is not abstract and not an interface.

Example

    class Animal {
        // Fields (Variables)
       String name;
    
        // Constructor
       Animal(String name) {
            this.name = name;
        }
    
        // Method (Function)
        void makeSound() {
            System.out.println(name + " is making a sound!");
        }
        
        public static void main(String[] args) {
            Animal dog = new Animal("Buddy"); // Creating Object
            dog.makeSound(); // Calling Method
        }
    }
output 
Buddy is making a sound!

‚úÖ It is a normal class ‚Äì Not abstract, not interface.
‚úÖ It contains complete methods ‚Äì No need for subclass implementation.
‚úÖ Objects can be created from it ‚Äì Unlike abstract classes.
‚úÖ It can extend other classes ‚Äì Inheritance works.
‚úÖ It can implement interfaces ‚Äì Supports multiple behaviors. 

A car has properties like brand, speed, and color. It also has actions like drive() and brake().
A concrete class is perfect for modeling a real-world object.
    class Car {
        // Fields
        String brand;
        int speed;
    
        // Constructor
        Car(String brand, int speed) {
            this.brand = brand;
            this.speed = speed;
        }
    
        // Method to drive the car
        void drive() {
            System.out.println(brand + " is driving at " + speed + " km/h");
        }
    
        public static void main(String[] args) {
            Car myCar = new Car("Toyota", 80); // Creating Object
            myCar.drive(); // Calling Method
        }
    }
output 
Toyota is driving at 80 km/h
