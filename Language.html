Language

Any language which is used to communicate with a machine or computer is called a programming language.

Types of Language

  • Low Level Language / Binary Language
  • Assembly Language
  • High Level Language

Low Level Language

A language which is written in binary form is called a low-level language, but it is difficult for humans to understand.

0 => Off | Example: HELLO
1 => On | H => 01001000
1 Byte => 8 bits

Assembly Language

A mid-level language is called assembly language. In assembly language, predefined words are used to make programming easier.

Assembly language is easier for humans, but a computer cannot understand it directly.

Examples:

Predefined Address of memory Assembler
ADD %221 %11b
DIV %11c %111
POP %11b LLB
PUSH %11c %411
CALL @11b 5656

Predefined + Address of memory => Assembler => CPU.

A computer cannot understand assembly-level language directly, so an assembler is used as a translator to convert assembly language into low-level language.

High Level Language

A language written in human-readable form is called a high-level language. High-level language is easy for humans to understand, but a computer cannot understand it directly.

Example=> C Language (1972)

A computer cannot directly understand high-level language, so a compiler or interpreter is used as a translator to convert high-level language into a lower-level language.

Compiler

A compiler is a translator that reads the entire code at once. If the code is correct, the compilation is successful.

Interpreter

An interpreter is a translator that reads the code line by line and generates low-level code one line at a time. If it encounters an error, execution stops at that line.

Note: A compiler provides faster execution than an interpreter.

Types of High-Level Language

  • Procedure-Oriented Language
  • Object-Oriented Language

Difference Between Procedure-Oriented and Object-Oriented Programming

  • Procedure programing language is not a secure programing language but Object oriented programing language is a secure programing language.
  • Class and Object is not uses in procedure programing language but class and Object use to design a Object oriented.
  • Procedural oriented programing language is plateform dependent but Object oriented is platefrom independent.
  • Procedural oriented is not supports OOPS But Object oriented is supported OOPS.
