Java is very easy to learn, and its syntax is simple, clean, and easy to understand. According to Sun Microsystems, Java language is a simple programming language because:
  • Java syntax is based on C++ (so easier for programmers to learn it after C++)
  • Java has removed many complicated and rarely-used features, for example, explicit pointer, operator overloading, etc.
  • There is no need to remove unreferenced objects because there is an Automatic Garbage Collection in Java.

1. Open-Source

A program in which the source code is available to the general public for use and/or modification from its original design at no cost is called open source.

2. Secured

Java is best known for its security. With Java, we can develop virus-free systems. Java is secured because:

  • No explicit pointers.
  • Java programs run inside a virtual machine sandbox.
  • Java provides automatic garbage collection which runs on the Java Virtual Machine to get rid of objects which are not being used anymore.
  • It provides a virtual firewall between the application and the computer, so it doesn't grant unauthorized access.

3. Portable

"Write once, run anywhere". Java code written on one machine can run on another machine. Java is portable because it facilitates you to carry the Java bytecode to any platform. It doesn't require any implementation.

4. Dynamic

Java is a dynamic language. It supports the dynamic loading of classes, meaning classes are loaded on demand. It also supports functions from its native languages, such as C and C++.

5. Distributed

Java is distributed because it facilitates users to create distributed applications in Java.

6. High-performance

Java is faster than other traditional interpreted programming languages because Java bytecode is "close" to native code.

Terminology

Java Development Kit (JDK)

The Java Development Kit (JDK) is a software development environment used to develop Java applications and applets. It physically exists. It contains JRE + development tools. The JDK is used for development purposes and includes the execution environment as well.

*JDK versions: 1.0 to 1.9.
*Mostly V1.8 is used now.
:JDK is java development kit which is for development purpose and it include execution envirnoment also.

Java Runtime Environment (JRE)

The Java Runtime Environment is a set of software tools used for developing Java applications. It provides the runtime environment for Java programs.
:JRE is java runtime environment It is predifined. Class-file,library files.

Java Virtual Machine (JVM)

It is mainly used to allocate the memory and compiling.
:JVM are not platefrom independent,JVM are platefrom specific run tine implimentation provide by the vandors.
:JVM is a purely runtime envirnoment and hence you will not be able to compile your source file using jvm.
:JVM (Java Virtual Machine) is an abstract machine. It is called a virtual machine because it doesn't physically exist. It is a specification that provides a runtime environment.

Just In Time (JIT)

Just In Time compilation is a process that improves the performance of Java applications by compiling bytecode into native machine code at runtime. This improves performance and ensures smooth operation.
:JIT compilation is enabled by default.
