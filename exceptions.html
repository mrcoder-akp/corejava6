Exception

It is abnormal condition faced by program during rum time.
Few abnormal-condition that a program can face as follow-
1.Divide by 0(Airthmatic execption)
2.File not found(filenotfound execption)
3.Database table does not exits(SQL-execption)
4.Array index out of range(Array index out of bound execption)
If program is not handdling-Execption then JVM will terminate execption of that program.
1.Protecting the program from bing terminated.
2.Information user about the execption
Due to abnormal condition following acitivites will be performed by JVM or Methods or Constructor
(Remember program is current using JVM or Method or Constructor to perform task)

1Creating exception
2.Throwing exception
3.Catching execepton

Creating exception

when Program(JVM predefine method or predefine construtor)encounter an abnormal condition then an object is created by JVM predefine method or predefine construtor.In this object information about that abnormal condition is stored.this object is called Execption and this process is called creating execption.

Throwing exception

After creating exception,JVM predefine method or predefine construtor throw that exception to our program this process is called throwing execption. actually reference of exception is sent to the program by JVM or API.

Catching exception

Our program will accept reference of exception this proccess is called catching exception.
If program will not accept this exception then exception will be accepted by JVM.In this case program will be terminated.

Exception Classes

These classes are in hierarchical manner.
Top most class of the hierarchy is Throwable class

Throwable class

Every exception class inherits this class directly or indirectly.It has following two child classes-
This class is super class of all exception classes.
This class has two sub classes-
1.Exception class
2.Error class

Exception class

This class has property of throwable class it means object of this class can be thrown.
It has handling exception property also.
Every exception class must inherited this class to acquring properties from this class.
Exception class can inherited throwable as well as catchable property from throwable classes.
Exception class can inherited catchable property from the exception class.
Remember Exception class is a child of throwable class.It means exception class has more property of throwable class.
It has several child classes.Few of them are as follows--
1.FileNotFoundException class
2.ClassNotFoundException class
3.IterruptedException class
4.IOException class
5.RuntimeException class
etc

RuntimeException class

It is a child of exception class
It has several child classes.Few of them are as follows
1.ArithmeticException
2.NullPointerException
3.ArrayIndexOutOfBoundsException
4.NumberFormatException
5.ClassCastException
Except RuntimeException class, other child classes of Exception class are checked exceptions
Exception class itself is checked exception
RuntimeException class and all childs of this class are unchecked exceptions.

Putting Handler

Source code of the program must have exception handler.This handler insert by you inside source code of the program.
Responsibility of this program--
1.Catching exception
2.Displaying exception message on screen.
you will have to define following two block.
Use following two blocks to catch exception
1.try block
2.catch block

try block

Inside this block, write those statements/commands that might throw exception

try { //statements/commands }

catch block

Declered argument of type crossponding exception class(Such as Airthmatic,filenotfound,Database,Arrayindex etc)
Inside body of catch,write,command to display exception message.
Just immediately after try block, write catch block as method

catch(ExceptionClass ex) { //command to display exception message }

Note:A try block can have multiple catch blocks.

finally block

This block is optional.
This block is always be executed wether exception is thrown or not and catch or not So Commands that must be executed in any condition, write inside this block.

try { // Code that may throw an exception } catch (ExceptionType e) { // Code to handle the exception } finally { // Code that will always execute }

Types of exceptions

Exceptions are divided into following two categories...
1.Checked exceptions
2.Unchecked exceptions

Checked exceptions

Precaution of these exception are not possible So putting exception handler(Try,catch) inside source code of the program must be mandatory.
This is responsible of compiler to chek and extansion of handler not found then compiler will genrate error.
Except Runtimeexception class and its child.
Handling these exceptions are mandatory
Compiler will check existense of handler(try and catch) inside source code of program.If not found then compiler will generate error.
Exception class and all child classes(Except RuntimeException class and childs of RuntimeException class) of Exception class are checked exception
These exceptions are also known as compile time exceptions

Unchecked exceptions

Precaution of these exception are possible, So putting exception handler(Try,catch) inside source code of the program is not mandatory.
Compiler will not chek existance of handler inside source code of the program.
Handling these exceptions are not mandatory.Because precaution of these exceptions are possible
RuntimeException class and all childs of RuntimeException class re unchecked exception
These exceptions are also known as runtime exceptions

File reading with Exception use
__________________________________
import java.io.FileNotFoundException;
import java.io.FileReader;
import java.io.BufferedReader;
import java.io.IOException;

public class App 
{
public static void main(String[] args) 
{
try 
{
FileReader fr=new FileReader("aa.txt")
//When a FileReader object is created, its constructor will be called, and the file will be loaded into memory.
// The data from the file will be stored in a memory container, i.e., an object in the FileReader class.
BuffereReader br=new BuffereReader(fr);
String str=br.readLine();//readLine() is a BuffereReader class and this method read data
while (str != null) 
{
System.out.println(str);
str = br.readLine(); 
}
br.close();
}
catch (FileNotFoundException ex) {
System.out.println("File not found: " + ex);
} 
catch (IOException ex) {
System.out.println("IO Exception: " + ex);
}
}
}

Throw

Throw is used inside the body of a method.
Throw is used to throw an exception.
Exception Object throw.
The throw keyword is used to create and throw an exception object.
Inside a method or a block of code, when you need to signal an error or unexpected condition.
Only one exception can be thrown at a time using throw

Throws

The throws keyword is part of a method signature
It informs the caller of the method that the method may throw specific checked exceptions
Checked exceptions are exceptions that must be handled using a try-catch block or declared with throws.
To handle checked exceptions like IOException, SQLException, etc.
throws only declares exceptions; it doesn't throw them. The actual exception is thrown using the throw keyword inside the method.
Exception object forword caller.
It is written inside protype of method or constructor
This statement is responsible to forward checked exception(created and thrown by method or constructor) to the caller
Throws statement is mandatory for those methods and constructor that has command to create and throw checked exceptions
