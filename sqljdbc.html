SQL(Structured query language)

SQL stands for structured query language
By using this language we can write SQL statements/commands
These statements/commands will be executed by database server
Database server will give the result of these commands
Client program can communicate with database server(MySql,Oracle,etc) by using SQL Client program will write SQL queries by using this language Client program will send these queries to the database server Database server will execute these queries Commands could be as follows-----------------
1.Command to create,drop,alter and use database
2.Command to create,drop,alter tables
3.Command to insert record into table
4.Command to delete record from table
5.Command to update record from table
6.Command to select record from table
etc

These commands are divided into following three categories
1.DDL(Data definition language):crete table,alter table,drop table
2.DML(Data manipulation language):insert record,delete record,update record
3.DQL(Data query language):select record

JDBC(Java Database Connectivity)

It is an API(Specially methods)
Our program will call this API to connect to the database server(such as MySql,Oracle,MS-SQL server,DB2.etc) and execute SQL queries.
This API is made by both java vendors and database server vendors.Prototype of these methods(API) are developed by java vendors and implementation of these methods are provided by database server vendors.These methods are declared inside interfaces by java vendors.

Interfaces by java vendors

1.Driver interface
2.Connection interface
3.Statement interface
4.PreparedStatement interface
5.CallableStatement interface
6.ResultSet interface
etc
java.sql package contains these interfaces.
Implementation of these interfaces are given by database server vendors

database server vendors(Steps to connect and execute query)

1.Load driver class
2.Create connection object
3.Create statement object
4.Execute SQL qeury
5.Close connection

Driver class

Every driver class is an implementation of java.sql.Driver interface.Name of driver class will be given by database server vendors

public class MyDriver implements java.sql.Driver { }

Driver interface has several methods.One of them is as follows-
public Connection connect(String url,Peropties p)
This method will be called to connect with database server
Code to call connect() method is already written
This code is written inside getConnection() method of java.sql.DriverManager class
Our program will call getConnection() method of DriverManager class and getConnection() method will call connect() method of driver class.
Remember DriverManager class must have reference of object of driver class
This is responsiblity of our program to provide reference of driver class to the DriverManager class.

Note

Every driver class on loading, creates an object of itslef and gives reference of this object to the DriverManager class" So our program must load driver class

Loading driver class

As we know, on loading, driver class creates an object of itself and gives reference of this object to the DriverManager class So our program must load driver class
Write following code to load driver class:
Class.forName("com.mysql.cj.jdbc.Driver");
//forName is a static method of java.lang.Class class(object class)
//This method is responsible to load specified class into memory
This code will load driver class and create an object of driver class and give reference of this object to the DriverManager class

static block

We can define this block inside a class public class AA { static { //command } } This block is executed when class is loaded in memory

Creating statement object and executing SQL queries

Our program will create this object to call methods of Statement interface or PreparedStatement interface These methods will be responsible to execute SQL queries
Inorder to execute SQL queries our program will call methods of the following interfaces.

Statement interface

Call methods of this interface when your program wants to execute SQL static query(non-parameterized queries).All DDL query are static select,delete,etc query without where are also static query.
Object of this interface must be created to call methods of this interface.
//Command to create object of Statement interface
Statement st=cn.createStatement();
Statement interface has several methods.Few of them are as follows--
1.public boolean execute(String query):Call this method to execute DDL query
2.public int executeUpdate(String query):Call this method to execute DML query
3.public ResultSet executeQuery(String query):Call this method to execute DQL query
//Command to create object of Statement interface
Statement st=cn.createStatement();
//Command to execute DDL query
create table productinfo(pid int primary key,name varchar(30),brand varchar(30),price int)

Executing select query

Call following method of Statement or PreparedStatement interface to execute select query
public ResultSet executeQuery(String query):It is a method of Statement interface
public ResultSet executeQuery():It is a method of PreparedStatement interface

ResultSet interface

As we can see, return type of executeQuery() method is ResultSet interface.
executeQuery() method creates an object of this interface and keeps resultset(set of records given by database server) into this object and returns reference of this object to the caller.
Our program(caller) can fetch data from this object by calling getter method of ResultSet interface

executeQuery()

The executeQuery() method, which belongs to the ResultSet interface, is responsible for sending the SQL query passed to it the database server. Once the query is processed, the database server returns a result. This result is stored in a ResultSet object, and a reference to this object is given back to the caller. Statement st=cn.createStatement(); ResultSet rst=st.executeQuery("select * from employeeinfo"); PreparedStatement ps=cn.prepareStatement("select * from employeeinfo where eid=?"); ResultSet rst=ps.executeQuery();

Moving cursor

As we know every resultset object maintains a cursor which is initially positioned just before the first record(at index -1)
Inorder to fetch any record/data from this object we will have to place cursor on that record
We can call one of following methods of ResultSet interface to move cursor
1.public boolean first()
2.public boolean next()
3.public boolean previous()
4.public boolean last()
5.public boolean relative(int n)
6.public boolean absolute(int n)
etc
Every resultset object is bydefault forward only so you can call only next() method in that case.Other methods will not perform task.
Inorder to call other methods you will have to make scrollable resultset object.
ResultSet.TYPE_SCROLL_INSENSITIVE,ResultSet.CONCUR_UPDATABLE

Relative(int n)

If the cursor is currently at position 3 and n=4 is given, the cursor will move to position 7 (3 + 4). It shifts the cursor by n positions from its current location.

Absolute(int n)

If n=2 is given, the cursor will move directly to position 2, starting from the beginning, regardless of its current position.
